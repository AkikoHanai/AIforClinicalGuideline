# 定性的SR完全自動化パイプライン

## ✨ **回答: はい、生成できます（ただし人間確認が必須）**

extracted.csv だけから、以下を**自動生成**できます：
- ✅ 定性的SRテーブル
- ✅ RC-5（推奨作成の経過）
- ✅ RC-1（推奨文草案）

**ただし**、複数の工程で **人間のジャッジメント** が必須です。

---

| フェーズ | タスク | 自動化度 | 人間の役割 |
|---|---|---|---|

## 📋 各フェーズの確認タグ

### Phase 1: エビデンス総体テーブル

```
出力ファイル: phase1_evidence_table_annotated.json
チェックリスト: phase1_checklist.md

✅ 自動生成（確認不要）
   - n_studies: 論文数を自動集計
   - rob_summary: バイアスリスク自動集計

🟡（要確認）- 以下を確認してください
   - effect_direction: LLMが抄録から推定
     確認項目: 複数研究間で矛盾がないか？
   
   - GRADE確実性: RoB結果から簡易推定
     確認項目: 形式的GRADE評価と乖離していないか？

⚠️ データ不十分
   - 効果量（元論文へのアクセスなし）
     改善案: 二次スクリーニング時にS3から元論文入手
```

### Phase 2: EtDフレームワーク

```
出力ファイル: phase2_etd_annotated.json
チェックリスト: phase2_checklist.md

✅ 自動生成（確認不要）
   - pico.O: アウトカム自動抽出

❌ 手動入力必須 - 以下は必ず入力してください
   - pico.P: 対象の詳細定義
     例: "18～64歳のがんが治癒・安定しているがんサバイバー"
   
   - pico.I: 介入の詳細定義
     例: "中強度以上の有酸素運動（週3-5日、30-60分）"
   
   - pico.C: 比較対照の定義
     例: "運動なし、または標準的ケアのみ"
   
   - values_preferences: 患者価値観データ
     データ源: 患者インタビュー、FGD等
   
   - resource_use: 医療経済情報
     確認項目: 費用、保険適用、診療報酬等
   
   - feasibility: 実装可能性
     確認項目: 施設、人員、地域差等

🟡（要確認）- レビューしてください
   - benefit_harm_balance: 益・害バランス
     確認項目: 『軽微な有害事象』と『重篤な有害事象』の重みづけは妥当か？
   
   - recommendation_strength: 推奨の強さ
     確認項目: LLM推定値は参考値のみ。最終決定はパネル投票で
```

### Phase 3: 推奨文生成（RC-5/RC-1）

```
出力ファイル: phase3_rc5_output.md
           phase3_rc1_draft.md

🟡（生成）Claude AI が自動生成
   ↓
🟡（要確認）専門家による確認が必須
   - RC-5内容: 医学的妥当性、根拠の正確性
   - RC-1表現: 推奨文の医学的正確性、臨床文脈への適合性
   
パネル投票（必須）
   - 推奨の方向: For / Against
   - 推奨の強さ: 強い推奨 / 弱い推奨
   - 合意度: 全員一致 / 大多数 / 意見分かれ
```

## 🚀 使用方法

### 1. 完全アノテーション版パイプライン実行

```bash
cd /Users/ahanai/SR

export ANTHROPIC_API_KEY="sk-ant-..."

python sr_fully_annotated_pipeline.py \
    --extracted ./sr_output/extracted.csv \
    --cq "CQ 1" \
    --outcomes "QoL, 筋力, 倦怠感, うつ" \
    --output-dir ./sr_output/annotated_sr_final
```

### 2. 出力ファイル確認

```
sr_output/annotated_sr_final/
├── phase1_evidence_table_annotated.json     ✅ 自動生成
├── phase1_checklist.md                      🟡 確認チェック
├── phase2_etd_annotated.json                ✅ 自動生成 + 🟡 確認
├── phase2_checklist.md                      ❌ 手動入力必須
├── phase3_rc5_output.md                     🟡（生成）
└── phase3_rc1_draft.md                      🟡（生成）
```

### 3. ステップバイステップ確認

**Day 1-2: Phase 1確認**
```
□ phase1_checklist.md を開く
□ 🟡（要確認）項目を確認
  - effect_direction の妥当性
  - GRADE確実性の妥当性
□ 必要に応じて修正
```

**Day 3-4: Phase 2入力**
```
□ phase2_checklist.md を開く
□ ❌（手動入力必須）を記入
  - PICO(P,I,C)の詳細定義
  - 患者価値観・経済・実装データ
□ 🟡（要確認）をレビュー
```

**Day 5: Phase 3確認+投票**
```
□ phase3_rc5_output.md を読む
□ phase3_rc1_draft.md を読む
□ パネル会議で投票
  - 推奨の方向
  - 推奨の強さ
  - 合意度
□ 最終確定
```

## 🔑 重要: タグの意味

### ✅ 自動生成
- 人間確認不要
- そのまま使用可能

### 🟡（要確認）
- LLMまたはヒューリスティックで生成
- 専門家による確認が推奨
- 修正が必要な場合がある

### 🟡（生成）
- Claude AIが自動生成
- 信頼度は限定的
- 最終確認が必須

### ❌（手動入力必須）
- 自動生成不可
- 人間が必ず入力する
- なしでは次のフェーズに進めない

### パネル投票
- ガイドラインの正式決定
- 専門家会議で実施
- 推奨の強さ・合意度が決まる

## 📊 全体フロー

```
extracted.csv
    ↓
【Phase 1】自動生成 (70%) + 確認 (30%)
  → phase1_checklist.md で 🟡（要確認）項目確認
    ↓
【Phase 2】自動生成 (40%) + 手動入力 (60%)
  → phase2_checklist.md で ❌（手動入力必須）を記入
  → 🟡（要確認）をレビュー
    ↓
【Phase 3】自動生成 (55%) + 確認 + パネル投票 (45%)
  → phase3 ファイルを確認
  → 推奨の方向・強さ・合意度を投票
    ↓
【完成】ガイドラインに統合
```
