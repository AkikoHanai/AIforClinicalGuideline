# 自動化到達度レポート

## 概要

**質問**: エビデンス集約（extracted.csv）だけから、定性的SR、RC-5、RC-1 の生成は可能か？

**答え**: 🟡 **部分的に可能だが、複数の工程で人間による確認が必須**

---

## 詳細：各ステップでの人間ジャッジメント

### 【Phase 1】定性的SRテーブル生成

#### 1.1 effect_direction の推定

```json
{
  "outcome": "QoL",
  "effect_direction": "改善"  ← 🟡 LLMが抄録から推定
                                  （根拠: SMD > 0 等の記述から）
                                  要確認: 複数研究間で矛盾がないか？
}
```

**自動化度合い**: 🟡 **70%** — 抄録の定性的記述から推定  
**手動確認が必要な場合**:
- SMD/RRが報告されていない論文が多い
- 「改善傾向」「有意性不明」など曖昧な記述
- メタアナリシス結果との乖離

#### 1.2 GRADE確実性の推定

```json
{
  "outcome": "QoL",
  "certainty_of_evidence": "Moderate",  ← 🟡 RoB結果から推定
                                            formula: High_count, Some_count から
                                            推定値のみ（正式なGRADE評価ではない）
}
```

**自動化度合い**: 🟡 **50%** — 簡易推定のみ  
**手動確認が必須な理由**:
- GRADEの正式評価は複数の要因（非直接性、非一貫性、不精確さ等）を総合的に判断
- 抄録からは「異質性」「出版バイアス」などを完全には判定不可
- 推定値は参考値に過ぎず、最終的には専門家パネルの判断が必要

### 【Phase 2】EtDフレームワーク生成

#### 2.1 PICO の定式化

```json
{
  "pico": {
    "P": "（対象の説明）",  ← ❌ 手動入力必須
         
    "I": "（介入の説明）",  ← ❌ 手動入力必須
    
    "C": "（比較対照の説明）",  ← ❌ 手動入力必須
    
    "O": ["QoL", "筋力", "倦怠感"],  ← ✅ 自動抽出
  }
}
```

**自動化度合い**: ❌ **0%** (P, I, C) / ✅ **100%** (O)  
**理由**:
- P（対象）: 「18～64歳のがんサバイバー」等、詳細定義はガイドラインの文脈から
- I（介入）: 「運動の具体的定義（有酸素/筋力/強度等）」は重要
- C（比較）: 「非介入」vs「標準的ケア」vs「別の運動」の区別が重要

**必須**: ガイドライン作成者による確認・修正

#### 2.2 益・害のバランス判定

```json
{
  "benefit_harm_balance": {
    "description": "望ましい効果（2アウトカム）が望ましくない効果（0アウトカム）を上回る",
    "judgement": "benefits exceed harms"  ← 🟡 自動判定
                                           ⚠️ 「筋肉痛」と「心血管イベント」を
                                           同じ「害」として扱うのは不適切
  }
}
```

**自動化度合い**: 🟡 **40%** — 数的判定のみ  
**手動確認が必須な理由**:
- 「軽微な有害事象」（筋肉痛）vs「重篤な有害事象」（死亡）の重みづけ
- 患者にとっての臨床的重要性の判断
- 文脈によって「害」の捉え方が異なる

#### 2.3 患者の価値観・医療経済・実装可能性

```json
{
  "values_preferences": {
    "description": "患者の価値観・嗜好性に関するデータは限定的",
    "certainty": "moderate uncertainty"  ← ⚠️ 「データ不十分」
  },
  "resource_use": {
    "description": "データ不十分",  ← ⚠️ 自動記載（推奨: 手動補足）
    "certainty": "insufficient data"
  }
}
```

**自動化度合い**: ❌ **0%** — 完全に手動補足が必要  
**手動確認項目**:
- 患者インタビュー・FGD等の一次データが必要な場合がある
- 医療経済評価の有無を確認（抄録には記載されないことがほとんど）
- 国内の実装可能性（施設、人員、費用等）

### 【Phase 3】RC-5 / RC-1 生成

#### 3.1 RC-5（推奨作成の経過）

```markdown
## 1. 臨床疑問の定式化（PICO）とアウトカムの選定

CQ 1: 運動習慣のない18～64歳のがんサバイバーに運動を勧めることが推奨されるか？

対象: 18～64歳の、がんが治癒・安定しているがんサバイバー
介入: 中強度以上の運動（週3-5日、30-60分）
比較対照: 運動なし
主要アウトカム: QoL, 筋力, 倦怠感, うつ, 運動関連有害事象

      ↓ 🟡 根拠テキストは限定的（抄録から）
         要確認: 全文論文と比較して過不足がないか？
```

**自動化度合い**: 🟡 **60%** — Claude が抄録から推定  
**手動確認が必須な理由**:
- 全文論文にしか記載されない詳細情報（サブグループ、タイムライン等）
- 抄録では省略された重要な限定条件
- メタアナリシスの異質性要因の詳細

#### 3.2 RC-1（推奨文草案）

```markdown
# 推奨文草案（RC-1: Individual Perspective）

運動習慣のない18～64歳のがんサバイバーにおいて、
運動を勧めることを提案する。
（推奨の強さ: 弱、エビデンスの確実性: C）

      ↓ 🟡 LLM生成（完全自動ではない）
         要確認: 
         - 推奨の強さは「弱」「中」「強」のどれか？
         - 日本の臨床文脈に合致しているか？
         - ガイドラインの他推奨との整合性は？
```

**自動化度合い**: 🟡 **50%** — Claude が EtD + RAG から生成  
**手動確認が必須な理由**:
- 推奨の強さの決定（panel discussion が通常）
- 国内医療体制への適応可能性
- 他のCQとの整合性確認

  }

## 結論

### 現状

✅ **エビデンス集約から自動生成は可能**

ただし、**完全自動化ではなく、複数の工程で人間による確認が必須**

### 自動化度合い

- **Phase 1 定性的SRテーブル**: 🟡 **70%** 自動化
- **Phase 2 EtDフレームワーク**: 🟡 **40%** 自動化
- **Phase 3 推奨文（RC-5/RC-1）**: 🟡 **55%** 自動化
- **全体**: 🟡 **50%** 自動化

### 推奨ワークフロー

```
extracted.csv
    ↓
【自動】定性的SRテーブル生成
    ↓
【確認】effect_direction, GRADE推定値 → 🟡（要確認）
    ↓
【手動】PICO (P,I,C) 定義、患者の値観・経済データ補足
    ↓
【自動】RC-5/RC-1生成
    ↓
【確認】臨床的妥当性、推奨の強さ決定 → 🟡（要確認）
    ↓
【完成】ガイドラインに統合
```
